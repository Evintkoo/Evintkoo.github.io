# Evint Leovonzko Data Science Portfolio

This is a modern, responsive portfolio website showcasing data science research and machine learning projects. The site features a professional dark mode design and interactive elements.

## Project Overview

**Owner**: Evint Leovonzko - Data Scientist  
**Purpose**: Portfolio website displaying research publications, business projects, and technical skills  
**Technology**: Pure HTML, CSS, JavaScript (no frameworks)  
**Design**: Dark theme with modern UI/UX, responsive across all devices  

## Technical Stack

- **Languages**: HTML5, CSS3, ES6+ JavaScript
- **Styling**: Custom CSS with CSS Grid, Flexbox, CSS Custom Properties
- **Fonts**: Inter (UI), JetBrains Mono (code)
- **Icons**: Inline SVG
- **Deployment**: GitHub Pages
- **Browser Support**: Modern browsers with CSS Grid support

## Development Guidelines

### Code Style
- **Consistent Naming**: BEM-like CSS, camelCase JavaScript
- **Modular Structure**: Separated concerns across files
- **Comments**: Comprehensive documentation in code
- **Error Handling**: Graceful fallbacks for unsupported features

### Content Management
- **Research Papers**: Individual HTML files with detailed methodology
- **Project Cards**: Standardized format with tech stacks
- **Contact Links**: External links with proper rel attributes
- **Assets**: Optimized images with proper alt text

### Performance Considerations
- **CSS**: Efficient selectors, minimal reflows
- **JavaScript**: Event delegation, minimal DOM manipulation
- **Images**: Optimized file sizes, proper formats
- **Fonts**: Efficient loading with display:swap

## Customization Notes

### Adding New Projects
1. Add project card to appropriate category in index.html
2. Include tech stack tags for consistency
3. Ensure proper external link attributes
4. Update meta descriptions if needed

### Styling Updates
- Use CSS custom properties for consistent theming
- Maintain responsive breakpoints
- Test animations across devices
- Validate accessibility compliance

### Content Updates
- Research papers require individual HTML files
- Maintain consistent citation format
- Update project descriptions with metrics
- Keep contact links current

## Browser Compatibility

- **Modern Browsers**: Full feature support
- **CSS Grid**: Required for layout
- **ES6 Features**: Arrow functions, const/let, template literals
- **Fallbacks**: Graceful degradation for older browsers
